# AgentHub 3 分钟流程讲稿

本文档用于快速介绍 AgentHub 的主要使用流程。目标是在 3 分钟内覆盖对话列表、单聊、群聊、PMO 主 Agent、多 Agent、产物预览编辑、Diff、版本历史、部署和上下文管理。

## 1. 开始前检查

```bash
npm run dev:all
```

打开本地地址后确认：

- 可以进入登录页和主工作台。
- 左侧导航和会话列表正常显示。
- 右侧工作台可以切换 Preview、Code、Diff、PPT、History、Deploy、Context。
- 模型、默认服务器和 CLI 平台在设置页显示真实可用或待配置状态。

## 2. 时间轴

| 时间 | 画面 | 讲解 |
| --- | --- | --- |
| 0:00 - 0:20 | 登录页与品牌入口 | “这是 AgentHub，一个多 Agent 协作平台。它把需求、Agent 调度、产物和部署放在同一个工作台里。” |
| 0:20 - 0:40 | 工作台、左侧导航和会话列表 | “左侧是 IM 式会话列表，支持单聊、群聊、置顶、归档、搜索和 Agent 能力标签。” |
| 0:40 - 1:05 | 切换 Agent 单聊和协作群聊 | “单聊适合让某个 Agent 专注执行，群聊适合复杂任务，由 PMO 主 Agent 统一协调。” |
| 1:05 - 1:30 | 展示群聊智能体启用/静音和任务面板 | “多人讨论时 Agent 默认静音，群主确认后启用智能体，PMO 只读取启用区间内的上下文。” |
| 1:30 - 1:55 | 展示聊天中的产物卡片和右侧 Preview | “Agent 的回复不是静态文本，网页、文档、PPT、代码都会进入右侧工作台继续操作。” |
| 1:55 - 2:15 | 切到 Code、Diff、History | “代码可以编辑，Diff 可以对比和应用，版本历史可以回看 v1/v2，满足产物编辑和二次迭代。” |
| 2:15 - 2:35 | 切到 Deploy 面板 | “部署卡片展示构建、上传、发布、日志和访问链接；未配置的平台会明确显示待配置或失败原因。” |
| 2:35 - 2:50 | 展示 Context、Workflows、My Agents | “用户可以把消息、文档段落或代码片段加入上下文，也可以引用工作流、创建自己的 Agent。” |
| 2:50 - 3:00 | 回到群聊总结 | “这条链路覆盖了交互体验、主 Agent、多 Agent 接入、产物预览编辑和部署能力。” |

## 3. 讲解重点

- 协作：PMO 负责拆解和协调，子 Agent 负责执行。
- 产物：网页、文档、PPT、代码、Diff、版本和部署都能在工作台内继续操作。
- 真实边界：CLI、模型 API、部署密钥未配置时会展示真实状态，不会包装成成功。

## 4. 备用讲法

如果模型接口临时不可用：

> “模型通过统一 adapter 层接入，当前状态会在设置页标成待配置或失败。产品链路仍可通过已有会话、产物工作台、部署状态和上下文管理说明，但不会把失败包装成成功。”

如果被问到密钥：

> “密钥只通过本地 `.env`、服务器 `.env` 或部署平台密钥配置，仓库提交中不会包含真实 Key。代码和文档只保留环境变量名。”

## 5. 录制建议

- 浏览器宽度建议保持 1440px 以上，三栏结构更清楚。
- 鼠标动作放慢，不要频繁来回切换。
- 每个面板只停留 3 到 5 秒，重点讲“它对应哪条核心能力”。
- 录制前关闭无关浏览器标签和通知。
- 如果使用真实接口，先准备一条短需求，避免等待时间过长。

## 6. 结尾总结

推荐最后一句：

> “AgentHub 的核心不是把多个 Agent 名字放在一起，而是让它们在一个 IM 式工作台里完成从需求、调度、产物、编辑到部署的闭环。”
