# AgentHub 产品文档

## 1. 产品定位

AgentHub 是一个面向多 Agent 协作的软件生成工作台。它以 IM 会话为入口，让用户在单聊或群聊中发起需求；PMO 主 Agent 负责理解目标、拆解任务、调度 Codex、Claude Code 和用户自建 Agent，最终把代码、网页、文档、PPT、部署状态等产物沉淀到可预览、可编辑、可继续交互的工作台中。

一句话定位：

> AgentHub = 轻量 IM 协作入口 + PMO 主 Agent 调度 + 多 Agent 产物工作台。

## 2. 用户与场景

| 用户 | 主要诉求 | AgentHub 对应能力 |
| --- | --- | --- |
| 普通使用者 | 用自然语言提交需求并查看结果 | AI 智能助手、会话消息、任务确认、移动端轻量模式 |
| 项目负责人 | 看清任务进度、参与成员和交付状态 | PMO 调度、任务面板、状态条、部署卡片 |
| 开发者 | 让代码 Agent 生成、修改、对比和部署产物 | Codex/Claude Code CLI 入口、代码卡片、Diff、版本历史 |
| 团队成员 | 在真人讨论和 Agent 执行之间切换 | 通讯录、成员邀请、群主启用/静音智能体、上下文过滤 |
| 高级用户 | 接入自己的模型和工具链 | 自建 Agent、OpenAI 兼容 API、DeepSeek、MCP 服务、默认服务器部署 |

## 3. 核心用户流程

1. 用户登录 AgentHub，进入工作台。
2. 用户创建单聊或群聊，选择真实用户和智能体成员。
3. 群聊中默认可以先由真实用户自由讨论；当真实用户数量大于等于 2 时，群主可以启用或静音智能体。
4. 群主启用智能体后，系统记录 `[AGENT_START]` 边界；再次静音时记录 `[AGENT_END]` 边界。Agent 只读取启用区间内的上下文，界面渲染时隐藏内部边界标记。
5. 用户提出需求，例如“生成一个轻量图书管理系统并部署到默认服务器”。
6. PMO 主 Agent 拆解任务，按能力派给 Codex、Claude Code 或用户自建 Agent。
7. Agent 的普通说明以消息正文呈现；代码、网页、文档、PPT、Diff、部署链接等进入独立产物卡片。
8. 右侧产物工作台按当前会话和当前话题同步产物，支持代码编辑、预览、PPTX 下载、文档下载、Diff、版本历史和部署。
9. 用户可以引用消息、文档段落、代码片段或已保存工作流，继续交给群内指定 Agent 处理。
10. 用户触发部署后，部署状态在同一张部署卡片中更新，成功后展示访问链接，失败时展示原因并支持交给 Codex 继续修复。

## 5. 配置边界

AgentHub 区分“产品入口已实现”和“运行环境已配置”：

- Codex 和 Claude Code 已接入 CLI 适配入口，真实执行依赖服务器已安装对应 CLI 并配置可执行路径或环境变量。
- OpenAI、DeepSeek、火山引擎豆包等模型通过 OpenAI 兼容接口接入，真实 Key 只放本地或服务器环境变量。
- 默认服务器部署依赖 `SELF_HOSTED_SSH_HOST`、`SELF_HOSTED_SSH_USER`、`SELF_HOSTED_SSH_KEY`、`SELF_HOSTED_PUBLIC_URL` 等服务器环境变量。
- Vercel、Miaoda 等外部平台未配置密钥时会显示待配置或失败原因，不会伪造成成功状态。
- `.env`、API Key、SSH 私钥、部署输出和日志不进入 Git 仓库。

## 6. 体验原则

- 信息结构优先：左侧导航、会话列表、主聊天区、右侧工作台职责清晰。
- 控制复杂度：高级能力放在需要时出现，默认界面保持简洁。
- 克制使用品牌色：蓝紫只用于选中态、按钮、状态点和关键强调。
- 保持 IM 熟悉度：头像、时间、标签、消息操作入口要轻，避免大面积营销式卡片。
- 产物必须可继续操作：预览只是第一步，后续要能编辑、下载、对比、回滚和部署。
- 让 PMO 可解释：复杂任务应留下“为什么派给谁、当前状态是什么、失败如何降级”的痕迹。

## 7. 功能完成度

- 已覆盖对话列表、单聊、群聊、主 Agent 调度、多 Agent 成员、自建 Agent、消息操作和上下文管理。
- 已覆盖网页、代码、文档、PPT、Diff、版本历史和部署状态卡片。
- 已覆盖默认服务器和用户自有服务器部署配置，支持部署连接测试、部署日志和失败修复交接。
- 已覆盖知识库、文件中心、工作流引用、MCP 配置、通讯录邀请、设置自检和移动端轻量模式。
- 系统对需要外部环境的能力展示真实状态，未配置时返回明确的待配置或失败原因。

## 8. 后续优化路线

- 强化多人协作通知：邀请、入群、群主操作和 Agent 状态变更可以进一步接入实时通知。
- 强化代码冲突处理：将当前 Diff 和版本能力扩展为更完整的 Git 冲突检测与合并策略。
- 强化知识库检索质量：增加文档重切片、向量维度迁移提示、引用来源高亮和 token 预算。
- 强化部署生产化：把前端、后端、数据库、对象存储和部署目标拆分，并引入 HTTPS、域名和密钥托管。
