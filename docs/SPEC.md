# AgentHub Product Spec

## 1. 产品定位

AgentHub 是一个面向多 Agent 协作的软件生成工作台。它以聊天为入口，让用户像在 IM 中沟通一样发起需求；平台中的 PMO 主 Agent 负责理解目标、拆解任务并调度 Codex、Claude Code 和用户自建 Agent 完成代码、文档、网页、PPT、部署等工作。

一句话定位：

> AgentHub = 轻量客服台式聊天体验 + 多 Agent 协作中枢 + 产物预览编辑工作台。

## 2. 用户与场景

| 用户 | 主要诉求 | AgentHub 对应能力 |
| --- | --- | --- |
| 产品/课程答辩者 | 快速展示多 Agent 协作闭环 | 验收指南、演示会话、PMO 调度、产物预览 |
| 开发者 | 让不同代码 Agent 协同实现需求 | Codex/Claude Code 联系人、能力标签、Diff 与版本 |
| 项目负责人 | 看清任务进度、风险和部署状态 | 任务面板、部署卡片、上下文引用、状态徽标 |
| 普通用户 | 用自然语言提出修改需求 | 单聊、群聊、文档段落引用、二次交互 |

## 3. 核心用户流程

1. 用户登录 AgentHub，进入工作台。
2. 用户从会话列表进入单聊或群聊。
3. 用户提出需求，例如“生成一个活动页并部署”。
4. PMO 主 Agent 分析需求，拆成设计、代码、检查、部署等任务。
5. Codex、Claude Code 或自建 Agent 并行处理任务。
6. 聊天流中出现代码 Diff、网页预览、文档、PPT、部署状态卡片等产物。
7. 用户在右侧工作台预览或编辑产物，也可以引用段落或代码片段再次交给 Agent。
8. 用户确认后触发部署，并查看部署日志和访问链接。

| 课题要求 | AgentHub 当前实现 | 演示入口 |

## 5. 体验原则

- 信息结构优先：左侧导航、会话列表、主聊天区、右侧工作台职责清晰。
- 克制使用品牌色：蓝紫只用于选中态、按钮、状态点和关键强调。
- 保持 IM 熟悉度：头像、时间、标签、消息操作入口要轻，避免大面积营销式卡片。
- 产物必须可继续操作：预览只是第一步，后续要能编辑、对比、回滚和部署。
- 让 PMO 可解释：每个复杂任务都应该留下“为什么派给谁、当前状态是什么”的痕迹。

## 6. 验收标准

答辩时至少证明以下能力：

- 能创建或切换多个会话，包含单聊和群聊。
- 群聊中能看到 PMO、Codex、Claude Code 与自建 Agent 的协作边界。
- Agent 联系人有头像、名称、能力标签和状态。
- 聊天消息中能出现部署卡片和产物卡片。
- 右侧工作台能预览网页、文档、PPT 和代码。
- 能查看代码 Diff 和版本历史。
- 能把消息或产物内容加入上下文，继续交给 Agent。
- 能说明真实适配器、系统托管能力和待配置状态的边界。

## 7. 后续优化路线

P0 已覆盖答辩主链路：聊天、多 Agent、产物、部署和运行自检。

P1 建议继续加强：

- 更完整的消息操作：回复、引用、复制、重新生成、收藏、锁定上下文。
- 主 Agent 调度细节：展示依赖关系、并行队列、失败原因和接管记录。
- 真实连接状态：为 Codex、Claude Code 和自建 Agent 展示可用、待配置、失败、降级等状态。
- 产物选择交互：用户框选文档段落或代码行后，一键派给指定 Agent。

P2 可作为创新点：

- 视觉化 Agent 工作流图。
- 多端部署目标，如 Vercel、飞书妙搭、静态对象存储。
- 自动生成答辩视频脚本和验收报告。
- 从历史会话中自动总结项目知识库。
