# AgentHub 结题验收清单

这份清单用于最后一次答辩前检查，目标是确认“能演示、能解释、能恢复”。

## 1. 启动与环境

- `docker compose up -d postgres redis` 能启动依赖。
- `npm run dev:all` 能启动前端和后端。
- `http://localhost:3000` 能打开工作台。
- `http://localhost:3002/api/health` 返回 `status: ok`。
- 设置页“结题演示自检”能显示后端、模型、部署服务器和工作区状态。

## 2. 课题要求覆盖

| 课题要求 | 必须展示的画面 | 通过标准 |
| --- | --- | --- |
| 对话列表 | 会话页左侧列表 | 能看到单聊、群聊、置顶、归档、搜索和标签 |
| 单聊模式 | 任一 Agent 单聊 | 能发送消息并得到 Agent 回复 |
| 群聊模式 | PMO + 多 Agent 群聊 | PMO 能拆解任务，多个 Agent 能参与 |
| 部署状态卡片 | 聊天流或部署面板 | 部署中和部署完成在同一卡片状态里呈现 |
| 产物预览 | 右侧 Preview / PPT / 文档 | 网页、文档、PPT 或代码能进入工作台 |
| 产物编辑 | Code / Diff / History | 能查看代码、对比变更、保留版本历史 |
| 二次交互 | 消息操作与上下文 | 能引用消息、文档段落或工作流继续处理 |
| 三方平台部署 | Deploy 面板 | 能生成外部访问链接或解释部署边界 |
| 主 Agent 概念 | 任务面板 | PMO 的拆解、调度、汇总过程可见 |
| 多 Agent 接入 | 智能体市场 / 我的智能体 | 至少展示 Codex、Claude Code、Open Code 或自建 Agent |

## 3. 会话核心体验

- 用户发送消息后，普通文字以流式体验出现。
- 代码内容进入代码卡片，不混在正文里。
- 产物工作台能跟随当前话题同步，不串台。
- 群聊中 2 个及以上真实用户时，只有群主能启用或静音智能体。
- 智能体静音区间不会被 Agent 读取，启用区间可作为上下文。
- 切换会话不会丢失消息，也不会把其他会话产物带过来。

## 4. 部署链路

- 默认服务器在设置页和产物工作台显示一致。
- 部署失败能看到失败原因，下一次任务开始后失败状态能自然消失。
- 部署成功后能拿到访问 URL。
- 同一产物不同版本部署后，访问链接能对应最新版本。

## 5. 自建智能体

- 能创建自建智能体。
- 能填写能力标签、系统提示词、模型、Base URL 和 API Key。
- 自建智能体能被加入群聊。
- 执行失败时有可解释提示，而不是静默卡住。

## 6. 移动端

- 手机端不展示复杂工作台，只保留 AI 助手、会话查看、消息发送和任务确认。
- 手机端不会因为 `crypto.randomUUID` 兼容性问题白屏。
- 电脑端仍作为主要编辑、预览和部署终端。

## 7. 隐私与仓库

- `.env.local` 未被跟踪。
- `deploy-output/`、SSH 私钥、数据库文件、日志和临时截图未被提交。
- 远端推送到 Gitee：`git@gitee.com:yellow-family-hero/agenthub.git`。
- 仓库中只保留环境变量名和示例值，不包含真实 API Key。

## 8. 答辩备用话术

如果模型接口临时不可用：

> 模型接入通过统一配置层完成，接口不可用不会影响产品主链路展示；PMO 调度、产物工作台、上下文管理和部署状态都可以独立验证。

如果被问到真实部署：

> 当前默认服务器用于答辩演示和轻量静态产物发布，生产环境可把后端、数据库、对象存储和部署目标拆开，并通过环境变量或部署平台密钥托管。

如果被问到多用户群聊：

> 真实用户自由讨论时 Agent 默认静音；群主启用后才开始读取启用区间内的上下文，避免用户未定稿内容误触发任务。
