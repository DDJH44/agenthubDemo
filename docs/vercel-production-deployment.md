# AgentHub Production Deployment

AgentHub should be deployed as two services:

1. The Next.js frontend runs on Vercel.
2. The AgentHub backend runs on your own server because it owns WebSocket sessions, file uploads, Prisma, SSH deployment, and long-running Agent jobs.

普通用户不需要配置 `.env`。`.env` 只属于平台管理员和服务器进程；用户在网页里登录后直接使用默认模型和默认部署服务器。

## Vercel Frontend

Create a Vercel project from this repository and use the root directory as the app root.

Build command:

```bash
npm run build
```

Runtime environment variables:

```bash
NEXT_PUBLIC_API_URL=https://api.your-domain.com
NEXT_PUBLIC_WS_URL=wss://api.your-domain.com/api/ws
```

Only `NEXT_PUBLIC_*` values are exposed to the browser. Do not put `OPENAI_API_KEY`, SSH keys, database passwords, or deployment secrets in Vercel frontend variables.

## Backend Server

Run `packages/server` on your own server, for example behind Nginx or Caddy.

Required environment variables:

```bash
PORT=3002
NODE_ENV=production
CORS_ORIGIN=https://your-vercel-app.vercel.app
DATABASE_URL=postgresql://agenthub:password@db-host:5432/agenthub?schema=public
SESSION_SECRET=replace-with-a-long-random-secret

ADAPTER_TYPE=openai
OPENAI_API_KEY=your-platform-llm-key
OPENAI_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
LLM_MODEL=your-volcengine-endpoint-id

UPLOAD_DIR=/var/lib/agenthub/files
MAX_FILE_SIZE_MB=100
DEPLOYMENT_TARGET_SECRET=replace-with-another-long-random-secret
```

Default deployment server for all users:

```bash
SELF_HOSTED_SSH_HOST=your-server.example.com
SELF_HOSTED_SSH_PORT=22
SELF_HOSTED_SSH_USER=deploy
SELF_HOSTED_SSH_KEY=/home/agenthub/.ssh/agenthub_default
SELF_HOSTED_DEPLOY_PATH=/var/www/agenthub-sites/{userId}/{deployId}
SELF_HOSTED_PUBLIC_URL=https://sites.your-domain.com/{userId}/{deployId}
```

Optional:

```bash
SELF_HOSTED_POST_DEPLOY_COMMAND=sudo -n nginx -s reload
VERCEL_TOKEN=your-vercel-token-for-user-requested-vercel-deploys
TAVILY_API_KEY=your-search-key
E2B_API_KEY=your-e2b-key
CLAUDE_CODE_PATH=claude
CODEX_PATH=codex
```

## Database

Use PostgreSQL with `pgvector` enabled. The schema includes vector embeddings, so a plain PostgreSQL database without the extension will fail knowledge-base operations.

On the database:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

Then run migrations from the backend environment:

```bash
npx prisma migrate deploy
```

## Reverse Proxy

Expose the backend as `https://api.your-domain.com` and make sure WebSocket upgrade is enabled for `/api/ws`.

Minimal Nginx shape:

```nginx
server {
  server_name api.your-domain.com;

  location / {
    proxy_pass http://127.0.0.1:3002;
    proxy_http_version 1.1;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
  }

  location /api/ws {
    proxy_pass http://127.0.0.1:3002;
    proxy_http_version 1.1;
    proxy_set_header Upgrade $http_upgrade;
    proxy_set_header Connection "upgrade";
    proxy_set_header Host $host;
  }
}
```

## User Model

Platform default configuration:

- The platform LLM key is configured on the backend.
- The default self-hosted deployment target is configured on the backend.
- Users can deploy without touching SSH keys or `.env`.

User personal configuration:

- User-owned LLM keys and deployment targets are saved through the UI.
- Private keys are generated by the backend and stored encrypted.
- The frontend only receives public metadata and generated public keys.

## Production Smoke Test

1. Open the Vercel frontend and log in.
2. Confirm the browser calls `NEXT_PUBLIC_API_URL`, not `localhost:3002`.
3. Confirm WebSocket connects to `NEXT_PUBLIC_WS_URL`.
4. Create a group chat and send a task.
5. Upload a small file and check it appears in Files and the conversation context.
6. Generate a simple HTML artifact and preview it.
7. Deploy it to the default server and open the returned URL.
